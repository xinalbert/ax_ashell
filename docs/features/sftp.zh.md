# SFTP

## 打开 SFTP 页面

从当前 SSH 会话打开 SFTP。页面同时提供远程文件浏览、本地文件浏览和该会话的传输状态。

## 会话 SFTP 路径

新建或编辑已保存的 SSH 会话时，可填写**SFTP 路径（可选）**，该会话打开 SFTP 后会直接进入对应远端目录。绝对路径会直接使用；`~` 和相对路径会基于服务器主目录解析。

未指定明确目标时，已保存会话会优先使用配置的**SFTP 路径（可选）**；没有配置时进入服务器主目录。从终端明确打开的路径优先级最高，并会在首次读取目录前解析，因此不会先闪过预设目录或 home。已打开的 SFTP 页面在空闲 worker 被回收后重新连接时，会恢复该页面当前显示的目录。

如需进入已连接 SSH 终端的当前目录，请使用远端地址栏旁的**打开终端当前目录**。它与远端地址栏手动输入的路径在创建 SFTP 连接时都会直接打开目标目录。在终端中执行 `cd` 不会再自动改变 SFTP 文件浏览器的位置。

## 本地目录记忆

可以在**设置 > 连接与网络 > SFTP > 默认本地目录**中设置全局默认目录。新建或未保存连接的 SFTP 页面会从这里开始。留空时使用用户主目录。

对于已保存的 SSH 会话，本地文件浏览器会优先恢复该会话最后一次成功打开的目录，而不是全局默认目录。如果已记忆的目录被删除或无法读取，AxShell 会打开默认本地目录，但不会覆盖原有记录。

这些本地路径只保留在当前电脑，不会进入 WebDAV 或 S3 会话同步。

## 文件操作

- 浏览远程目录，并显示或隐藏隐藏文件。
- 对已加载条目排序，并通过路径导航。
- 上传文件或文件夹。
- 下载文件或文件夹；目录下载会在适合时使用临时归档。
- 远端文件可以拖到本地面板下载。当前 GPUI 运行时尚不支持从 AxShell 向外拖到访达或资源管理器，包括已存在于本地面板中的文件。
- 新建文件夹并递归删除选中路径。
- 双击远端文件会创建受管本地编辑副本，并使用系统默认应用打开；双击目录进入目录；双击本地文件使用系统默认应用打开。
- 受管副本发生修改后会显示**完成编辑**。关闭编辑器后点击**完成编辑**，即可确认是否上传；关闭 SFTP 页面时也会先询问，避免遗留未处理的修改。
- **上传修改**会写回原远端路径；**保留本地副本**不会上传，副本会留在 AxShell 本地的 `sftp-edits` 目录中。
- 系统默认应用没有可靠、跨平台的“文档已关闭”事件。AxShell 不会把普通保存误判为退出；请在关闭编辑器后点击**完成编辑**。
- 预览支持的文件和受限目录内容。

## 超大目录

远程列表按需加载，不会一次把整个目录读入界面。

- 每页最多追加 250 个条目。
- 仍有条目时可以使用**加载更多**。
- 单个列表最多保留 2,000 个条目，或 2 MiB 的名称/路径数据。
- 达到安全预算后，AxShell 会停止加载并显示截断状态，不会把结果误报为目录结束。

## 传输任务

传输任务支持暂停、恢复和取消。传输历史显示进行中、完成、失败和中断状态，最多保留 100 条记录。

关闭 SFTP 页面会先询问已修改的受管编辑副本，随后再按已配置的确认流程处理活动传输，避免静默丢失工作。
